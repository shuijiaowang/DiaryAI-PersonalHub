# Windows 本地运行手册（不用 Docker）

> 假定你之前没装过 Python / Postgres / Node。一步步来，全部 PowerShell 命令。

---

## 0. 项目长什么样（先理解再动手）

```
DiaryAI-PersonalHub/
├── server/      ← 后端（Python）
│   ├── .env             ← 你要改的所有后端配置都在这里（密钥、数据库连接、模型）
│   ├── app/main.py      ← 后端入口
│   └── pyproject.toml   ← Python 依赖
├── web/         ← 前端（Vue 3 + TypeScript）
│   ├── .env.development ← 前端配置（默认指向 http://localhost:8000，一般不用改）
│   ├── src/main.ts      ← 前端入口
│   └── package.json     ← 前端依赖
└── docs/        ← 你看的这些文档
```

需要起两个进程：
- **后端**：`uvicorn` 跑在 http://localhost:8000
- **前端**：`vite` 跑在 http://localhost:5173

它们都需要 PostgreSQL 数据库（必装）。Redis 可选（不装也能跑）。

---

## 1. 安装三样东西

只装这三样，全程一次性。

### 1.1 Python 3.11+

下载：<https://www.python.org/downloads/windows/>
安装时务必**勾上 "Add Python to PATH"**。装完打开新的 PowerShell 验证：

```powershell
python --version
# 应该看到 Python 3.11.x 或更高
```

### 1.2 PostgreSQL 17（推荐）/ 16 也行

下载：<https://www.postgresql.org/download/windows/>（EnterpriseDB 安装包）
**直接选最新的 17，比 16 更省内存、JSONB 性能更好，本项目完全兼容。**
安装时记住三件事：
- **超级用户密码**（postgres 用户的密码）—— 自己记一个，随便比如 `postgres123`
- 端口默认 `5432`
- 安装完会附带一个 **pgAdmin 4** 图形界面，下面会用到

### 1.3 Node.js 20+

下载：<https://nodejs.org/>（选 LTS 版本）
装完验证：

```powershell
node --version    # v20.x.x
npm --version     # 10.x.x
```

> 不需要 pnpm，用 npm 就行。

## 2. 建数据库

打开 **pgAdmin 4** → 用安装时的密码登进去 → 左侧右键 `Databases` → `Create` → `Database…`，填：
- Database：`diary`
- Owner：`postgres`

点 Save。

> **更喜欢命令行？** 在 PowerShell 里：
> ```powershell
> # 假设 psql 在 PATH 里（默认安装会加）
> psql -U postgres -c "CREATE DATABASE diary;"
> # 输入安装时设的密码
> ```

接着我们要给应用用一个独立账号（**这一步可以跳过**，跳过的话直接用 postgres 超级用户也能跑，但不规范）：

在 pgAdmin 里点开数据库 `diary` → 顶部菜单 `Tools` → `Query Tool`，粘贴：

```sql
CREATE USER diary WITH PASSWORD 'diary';
GRANT ALL PRIVILEGES ON DATABASE diary TO diary;
ALTER DATABASE diary OWNER TO diary;
-- PostgreSQL 15+ 必须加这条，否则 diary 用户无法在 public schema 下建表
GRANT ALL ON SCHEMA public TO diary;
```

按 F5 执行。✅ 数据库准备好了。

## 3. 配置后端（最关键的一步）

打开 `server\.env.example` 复制一份成 `server\.env`：

```powershell
cd E:\code\DiaryAI-PersonalHub\server
Copy-Item .env.example .env
```

然后用编辑器（VSCode/Cursor）打开 `server\.env`，**只改这几个地方**：

```ini
# 1. 数据库连接（看你第 2 步是怎么建的）
# 如果用了独立账号 diary：
DATABASE_URL=postgresql+psycopg://diary:diary@localhost:5432/diary
# 如果直接用 postgres 超级用户（密码是 postgres123）：
# DATABASE_URL=postgresql+psycopg://postgres:postgres123@localhost:5432/diary

# 2. JWT 随便填一长串（生产环境必须改，本地随意）
JWT_SECRET=local-dev-secret-just-make-it-long-enough-please

# 3. 没装 Redis，关掉 LLM 缓存
LLM_CACHE_ENABLED=false

# 4. ★ DeepSeek API Key（必填，否则 AI 解析功能不能用）
DEEPSEEK_API_KEY=sk-你的 deepseek key
```

> **怎么拿 DeepSeek API Key？**
> 去 <https://platform.deepseek.com/> 注册账号 → 在 API Keys 页面生成一个 → 复制粘贴到上面。
> 充值最低 1 元起，足够你测试几百次。

## 4. 安装后端依赖 + 建表

依然在 `E:\code\DiaryAI-PersonalHub\server` 目录下：

```powershell
# 4.1 创建虚拟环境（让 Python 包只装在这个项目里，不污染全局）
python -m venv .venv

# 4.2 激活虚拟环境（PowerShell 专用）
.\.venv\Scripts\Activate.ps1
# 看到提示符变成 (.venv) PS E:\code\... 就对了
```

> **如果报错 "因为在此系统上禁止运行脚本"**，是 PowerShell 执行策略问题，开一个**管理员权限**的 PowerShell 跑一次：
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```
> 然后回到普通 PowerShell 重试激活命令。

```powershell
# 4.3 安装项目依赖（第一次会比较慢，一两分钟）
pip install -e ".[dev]"

# 4.4 让 Alembic 根据 models 自动生成第一份迁移脚本
alembic revision --autogenerate -m "init"

# 4.5 应用到数据库（真正建表）
alembic upgrade head
```

`alembic upgrade head` 没报错就说明 7 张表已经在 PostgreSQL 里建好了。可以去 pgAdmin 里 `diary` 数据库 → `Schemas` → `public` → `Tables` 刷新一下，应该能看到 user / diary / event / module / profile_section / ai_call_log / action_log 等。

## 5. 启动后端

依然在 `server` 目录、依然激活着 `(.venv)`：

```powershell
uvicorn app.main:app --reload --port 8000
```

终端会输出类似：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     [modules] registered builtin: weather
INFO:     [modules] registered builtin: meal
INFO:     [modules] registered builtin: expense
INFO:     [modules] registered builtin: memo
INFO:     [modules] sync done. registered codes: ["weather","meal","expense","memo"]
```

打开浏览器访问 <http://localhost:8000/docs>，能看到一份自动生成的 Swagger API 文档 = 后端 OK ✅

> **保留这个 PowerShell 窗口不要关**，后端要一直跑着。

## 6. 启动前端

**新开一个 PowerShell 窗口**（不要关掉后端那个）：

```powershell
cd E:\code\DiaryAI-PersonalHub\web

# 第一次需要装依赖（也比较慢，1~3 分钟）
npm install

# 启动开发服务器
npm run dev
```

终端会输出：

```
  VITE v5.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

打开浏览器访问 <http://localhost:5173>，应该看到登录页。

## 8. 平时怎么开发

每次开始干活只要做：

```powershell
# 窗口 1（后端）
cd E:\code\DiaryAI-PersonalHub\server
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --port 8000

# 窗口 2（前端）
cd E:\code\DiaryAI-PersonalHub\web
npm run dev
```

改后端代码 → uvicorn 自动重启
改前端代码 → vite 自动热更新

**改了 model（数据库表结构）之后**：

```powershell
# 在 server 目录，激活了 venv 的窗口里
alembic revision --autogenerate -m "describe what changed"
alembic upgrade head
```

## 9. 常见坑

| 症状 | 原因 / 解决 |
|---|---|
| `pip install` 提示 SSL 错误 | 公司/学校网络问题，临时换源：`pip install -i https://pypi.tuna.tsinghua.edu.cn/simple/ -e ".[dev]"` |
| `psycopg.OperationalError: connection refused` | Postgres 没启动。打开服务管理器（`services.msc`）找到 `postgresql-x64-17`（或 16）启动它 |
| `alembic upgrade` 报权限错误 | 你的 DB 用户对 `diary` 库没建表权限，回到第 2 步执行 `GRANT ALL` 那段 SQL |
| 后端能起，前端登录后 401 | Token 失效或后端重启了。前端按 F12 → Application → Local Storage 删掉 `diaryai.access_token` 重登 |
| AI 解析一直 `failed`，error 写 `DEEPSEEK_API_KEY is not configured` | `.env` 里 key 没填或填错 |
| AI 解析 `failed`，error 写 JSON 解析错误 | 模型偶尔抽风，会自动重试 3 次。还失败就再点一次「重新解析」 |
| 端口被占用 | `uvicorn` 改 `--port 8001`，然后改 `web\.env.development` 里的 `VITE_API_BASE` |
| 看不懂报错 | 把整段终端输出贴给 Cursor，让它帮你定位 |

## 10. 关掉一切

两个 PowerShell 窗口分别 `Ctrl+C`。
PostgreSQL 服务在系统启动时自动起，不管它就行。
