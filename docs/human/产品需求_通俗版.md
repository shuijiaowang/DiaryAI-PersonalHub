# 产品需求（通俗版）

> 1 页纸快速想起 MVP 在做什么。完整版/结构版在 `docs/ai/01_项目定位.md`。

## 是什么

一个只用"写日记"一种输入方式的个人数据中枢：

- 你每天写一段流水账自然语言；
- AI 自动把消费、饮食、运动、阅读、游戏、人际、情绪等切出来，存成结构化数据；
- 前端给你日/月/年的统计、备忘提醒、画像维护；
- 未来 AI 基于这份"画像"做主动建议、情感陪伴、自动化代办。

本质上是**个人 Agent 的入口**，不是日记 App。

## 为什么做

相信"软件被 AI 接管"是趋势，先做个最小雏形练手。
初期只是消费/饮食/备忘跑通；长期是任意模块（游戏、追番、健康…）都能接入；最终是外部数据也被动监听。

## 给谁用

阶段 1：自己一个人用（单用户、本地优先）
阶段 2：可注册多用户，彼此隔离

## MVP 要做到什么程度（验收场景）

> 打开网页 → 注册账号 → 在"我的资料"里填基本信息和女朋友昵称 → 在日记页粘贴一段真实日记（大概 6~10 行，含吃饭、消费、备忘）→ 点"AI 解析" → 几秒后看到三类事件被正确切出来 → 跳到"消费"页能看到当天 700+ 元的支出明细 → 跳到"备忘录"看到"洗澡 / 打扫 / 买抽纸"被自动添加为待办 → 第二天再写日记说"今天买了抽纸"，备忘录里那条自动被勾掉。

跑完 = MVP 成功。

## 不做的事（别劝）

- 不做社交 / 分享朋友圈
- 不做富文本编辑器（MVP 就是纯文本输入框）
- 不做复杂权限体系
- MVP 不接外部数据源（外卖、账单等远期再说）

## 几条设计原则（任何时候都别违反）

1. **AI 是后端的一部分**，可重试 / 可缓存 / 可审计 / 可回滚
2. **模块化优先**，加类型 = 加模块，不许改主流程
3. **原文即真理**，`raw_text` 永不丢
4. **可纠错**，单条重做 / 整篇回滚 / 用户改过的不被覆盖
5. **隐私**，字段级 privacy 控制是否进 AI 上下文
