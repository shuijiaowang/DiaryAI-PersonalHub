# docs/human · 给你看的文档

> 通俗、松散、以"1 分钟找回项目感觉"为目标。想让 AI 读的放隔壁 `docs/ai/`。

## 目录

| 文件 | 什么时候看 |
|---|---|
| `本地运行指南.md` | 换机器 / 重装环境 / 教别人跑 |
| `本地环境速记.md` | 自己查"我的密钥 / 版本号 / 常用命令" |
| `项目想法_原始.md` | 回忆最开始为什么做这个 |
| `产品需求_通俗版.md` | 一页纸快速想起 MVP 要做什么 |
| `数据库一览.md` | 知道有哪几张表、大概存什么 |
| `框架逻辑图解.md` | 一张图看清请求/解析是怎么流的 |
| `开发进度.md` | 自己记的里程碑 |
| `下一步建议.md` | 下一阶段想做什么（人工 + AI 建议） |
| `测试体验与反馈.md` | 跑起来之后的吐槽、bug、想改的地方 |

## 和 AI 目录的关系

- **同一件事只写一份细的**：数据库字段完整表在 `docs/ai/03_数据模型.md`；这里只有通俗概览。
- **AI 看 `docs/ai/`，你看这里**，但两边谁都允许过来翻——只是读起来密度不一样。
- **每个大阶段做完，AI 会按规则（见 `.cursor/rules/diaryai-rules.mdc` 第 15~17 条）同步 `开发进度.md` / `数据库一览.md` / `下一步建议.md`**。
