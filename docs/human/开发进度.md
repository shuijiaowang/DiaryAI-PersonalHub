## 里程碑

### 2026-04-20 · MVP 骨架端到端跑通

- 后端：FastAPI + SQLAlchemy + Alembic + 7 张表建齐
- 4 个内置模块注册：`weather / meal / expense / memo`
- AI 接入：DeepSeek via `LLMProvider` 抽象 + ai_call_log 审计
- 日记解析 pipeline：prompt 拼装 → LLM → schema 校验 → action 执行（带事务/回滚）
- 前端：Vue 3 + Naive UI，登录/注册/日记/详情/总览/模块明细/画像/设置页
- 本地闭环：真实 DeepSeek key 解析一段真实日记，能看到消费明细 + 备忘录待办

### 2026-04-20 · docs 目录重整

- `docs/ai/` 给 AI 用（结构化、短、token 友好）
- `docs/human/` 给自己看（通俗、随手翻）
- 新增 `.cursor/rules/diaryai-rules.mdc` 第 15~17 条，强制 AI 阶段性任务完成后同步文档

### 2026-04-20 · 消费统计独立页 + 手动补录（ADR-007）

- 新增"消费统计"独立页（侧边栏 / 统计 / 消费统计）
  - 日 / 周 / 月 / 年粒度切换 + 自由日期区间
  - 四块概览卡：总支出 / 笔数 / 单笔均值 / 最大单笔
  - 按时间段的柱+线组合图、按消费类型的饼图、全年日历热力图
  - 明细表支持分页，AI 提取 / 手动补录有视觉区分
- 新增「+ 手动补录」对话框：不修改日记原文，直接写 event，自动 locked=true
- ADR-007 定下来原则：日记原文永不被系统改写，用户补录一律走 event.locked
- Dashboard 的"近 30 天消费"卡暂时保留；后续如果用户只看独立页可以撤掉
