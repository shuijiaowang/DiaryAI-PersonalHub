# 数据库一览

> 只说"有哪几张表、大概存什么"。字段级细节在 `docs/ai/03_数据模型.md`。

## 7 张表

| 表 | 存什么 | 一句话 |
|---|---|---|
| `user` | 账号 | 用户名、邮箱、密码 hash |
| `diary` | 日记本体 | 一天一篇，原文 + AI 规范化文本 + 解析状态 |
| `event` | 解析结果 | AI 从日记切出的事件：属于哪个模块 + 原文片段 + 结构化 `data`（JSON） |
| `module` | 模块登记簿 | 注册了哪些"类型"（天气/饮食/消费/备忘…），enable 开关在这 |
| `profile_section` | 个人画像 | 模块化存储：基本信息/工作/人际/兴趣…，按 privacy 决定是否给 AI 看 |
| `ai_call_log` | AI 调用审计 | 每次调 LLM 都落一条：prompt、response、token、耗时、成本 |
| `action_log` | 动作执行日志 | AI 解析产出的 insert/update 每条都记录，用于回滚和纠错 |

## 图示（极简）

```
user ──< diary ──< event >── module
 │                   │
 │                   │
 └──< profile_section ┘  (都按 module_code 挂)

旁路：ai_call_log（挂 user+diary）   action_log（挂 diary）
```

## 几条不会变的规则

- **一天一篇日记**（`diary` 的 `(user_id, date)` 唯一）
- **原文永远保留**（`raw_text` 永不改）
- **用户改过的事件不会被 AI 覆盖**（`event.locked = true`）
- **时间全部 UTC 存**（`timestamptz`），前端自己转本地时区
- **半结构化数据用 JSONB**（`event.data` / `module.schema` / `profile_section.content`）

## 怎么改表

只走 Alembic，不允许手写 DDL：

```powershell
# 在 server/ 且 venv 激活
alembic revision --autogenerate -m "describe what changed"
alembic upgrade head
```

改完记得让 AI 同步更新本文件和 `docs/ai/03_数据模型.md`。
