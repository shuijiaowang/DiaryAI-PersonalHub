# 04 · 模块约定

## 三件套 + 一可选

每个模块必须提供：
1. **`code`** — 全局唯一短标识，英文小写（如 `expense`）
2. **`name`** — 中文名
3. **`DataSchema`** — Pydantic 模型，定义 `event.data` 的结构
4. **`prompt_fragment`** — 嵌入主 prompt 的描述片段
5. **`actions`** — 允许的 action 列表（默认 `["insert"]`；备忘录这种有状态的加 `update`）

可选：
- **`apply_action(action, db)`** — 默认行为是写 `event` 表；如需额外副作用（例如备忘录写 `profile_section`）才覆盖。

## 代码位置

```
server/app/modules/<code>/
├── __init__.py
├── schema.py          ← DataSchema (Pydantic)
├── prompt.py          ← prompt_fragment 字符串
└── module.py          ← 继承 _base.Module，用 @register 注册
```

注册入口：`server/app/modules/registry.py`。启动时 `main.py` 调用 registry 做 scan + DB sync。

## 新增模块的动作清单

1. 在 `server/app/modules/<new_code>/` 新建三件套文件
2. 在 `registry.py` 引入并 `@register`
3. 跑一次后端，确认启动日志里出现 `[modules] registered builtin: <new_code>`
4. 在 `server/tests/test_modules.py` 加最小用例：一段日记 → 断言生成了对应事件
5. **同步文档**：
   - `docs/ai/04_模块约定.md`（本文件）的"当前已注册内置模块"表里加一行
   - `docs/ai/07_进度与状态.md` 勾掉/新增 TODO
   - `docs/human/开发进度.md` 补一条里程碑
6. 跑 `pytest server/tests/` 全绿

## 手动补录（ADR-007）

- 用户前端"手动添加一条事件"走 `POST /api/v1/events`（schema: `EventManualCreate`）。
- Service 自动 `locked=True` + 归属到当天 diary（没日记就建一个 `raw_text=""` 的空壳）。
- 不写回 `diary.raw_text`，再解析不会覆盖 locked 事件。
- 所有模块共用这条通道，不需要每个模块自己写"手动补录"接口。

## 主流程禁区

以下文件不允许出现 `if module_code == "xxx"` 或按模块硬分叉：
- `app/ai/pipelines/diary_parser.py`
- `app/ai/pipelines/action_executor.py`
- `app/services/*`

任何模块特化行为都放进模块自己的 `apply_action`。

## prompt_fragment 写作要点

- 用精炼的一段中文 + 一段 JSON 示例，告诉 LLM 这个模块**什么时候该提取、字段含义**。
- 示例 JSON 必须能通过本模块的 `DataSchema` 校验（避免 LLM 学错）。
- 禁止在 fragment 里写"请遵守 JSON 格式"这种废话——主 prompt 已经有统一约束。
