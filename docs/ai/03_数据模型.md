# 03 · 数据模型

> 时间字段一律 `DateTime(timezone=True)`，业务全程 UTC。

## user
| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint pk | |
| username | varchar uniq | |
| email | varchar uniq | |
| password_hash | varchar | bcrypt |
| created_at / updated_at | timestamptz | |

## diary
| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint pk | |
| user_id | fk → user | |
| date | date | `(user_id, date)` uniq，一天一篇，重写视为追加/重解析 |
| raw_text | text | 原始日记（不可改写） |
| ai_processed_text | text | AI 规范化版本 |
| status | enum | `draft / parsing / parsed / failed` |
| parse_error | text | 失败原因 |
| created_at / updated_at | timestamptz | |

## event
| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint pk | |
| diary_id | fk → diary | |
| user_id | fk → user | 冗余加速查询 |
| module_code | varchar | `expense / meal / weather / memo / ...` |
| raw_text | text | 原文片段 |
| ai_processed_text | text | AI 规范化片段 |
| data | jsonb | 模块专属结构化字段（扁平 + `extra`） |
| locked | bool | 用户人工修正过 → AI 不能覆盖 |
| created_at / updated_at | timestamptz | |

索引：`(user_id, module_code, created_at)`；热字段可建 GIN（例如 `data->'price'`）。

## action_log
记录每个 `insert/update/delete` action 的执行结果，用于回滚与 AI 纠错。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint pk | |
| diary_id | fk → diary | |
| module_code | varchar | |
| action | varchar | `insert / update / delete / append` |
| target_id | bigint nullable | 作用于哪条记录 |
| payload | jsonb | action 原始内容 |
| status | enum | `success / failed / rolled_back` |
| error | text nullable | |
| created_at | timestamptz | |

## 迁移规则

- 改表只能经 Alembic：`alembic revision --autogenerate -m "..."` → 人工检查 → `alembic upgrade head`
- 禁止手写 DDL、禁止运行时 `Base.metadata.create_all`
- 改 schema 后必须同步 `docs/human/数据库一览.md`（一行人话描述）
