# docs/ai · 给 AI 看的上下文库

> 本目录是 AI 的"长期记忆"。文件刻意保持简短、结构化，降低每次读取的 token 成本。
> 给人看的通俗版在 `docs/human/`。硬约束在 `.cursor/rules/diaryai-rules.mdc`（始终加载）。

## 按任务选读（不要无差别全读）

## 文件清单

- `01_项目定位.md` — 一句话 + 核心名词（几行）
- `02_架构与目录.md` — 分层约束 + 目录骨架 + 关键文件指针
- `03_数据模型.md` — 表结构速查（字段级）
- `04_模块约定.md` — 模块三件套 + 当前已注册模块 + 新增步骤
- `05_核心流程.md` — 日记解析 / Action 执行 / 纠错回滚
- `06_工作流.md` — AI 接到任务的 SOP，含"改完必须更新哪些文档"
- `07_进度与状态.md` — 当前进度 / WIP / 未完 TODO / 已知 bug（动态维护）
- `08_决策记录.md` — 重要决策留痕，避免反复推翻

## 书写原则（维护本目录时遵守）

1. **单文件目标 ≤ 200 行**。超了就拆，或把论述性内容搬去 `docs/human/`。
2. **结论优先，论证后置**。AI 不需要被说服，只需要知道"就这么干"。
3. **表格、清单、代码块优先于长段落**。
4. **不写重复信息**。同一事实只在一处维护，其他地方链接过去。
5. **允许过时的部分要立刻删**，不要留"历史版本"段落。
