# 01 · 项目定位

**一句话**：以"每日日记"为唯一输入，AI 把自然语言切成结构化事件，沉淀为个人数字档案，并在其上提供统计、提醒、陪伴和自动化。

## 核心名词

| 名词 | 含义 | 对应表 |
|---|---|---|
| Diary | 一天一篇的原始日记 + AI 规范化文本 | `diary` |
| Event | AI 从日记切出的最小结构化数据单元 | `event` |
| Module | 事件/画像的类型单位，三件套 = schema + prompt_fragment + actions | `module` / `app/modules/<code>/` |
| Profile | 用户画像的模块化存储（给 AI 当上下文，也由 AI 更新） | `profile_section` |
| Action | AI 解析产出的"动作"（insert/update/…），由 ActionExecutor 执行 | `action_log` |

## 关键原则（产品层）

1. **AI 是后端的一部分**：调用必须可重试 / 可缓存 / 可审计 / 可回滚。
2. **模块化优先**：加类型 = 加模块，不动主流程。
3. **原文即真理**：`raw_text` 永不改写。
4. **可纠错**：单条事件重做 / 整篇回滚 / 用户修正后 `locked=true` 不被覆盖。
5. **隐私**：profile 字段级 privacy 控制是否进 AI 上下文。

## 范围边界

- ❌ 不做社交、富文本编辑器、复杂权限体系。
- ❌ MVP 不接外部数据源。
- ✅ MVP 目标：注册→写日记→AI 解析→看统计→备忘自动闭环。
