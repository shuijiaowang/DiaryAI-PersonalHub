# 02 · 架构与目录

## 技术栈（不讨论，就这么定）

- 后端：Python 3.11+ / FastAPI / SQLAlchemy 2.0 + Alembic / Pydantic v2
- 数据库：PostgreSQL 16+（JSONB；远期 pgvector）
- LLM：DeepSeek 默认，经 `LLMProvider` 抽象可换 OpenAI / Claude / Ollama
- 缓存/队列：Redis（可选，本地默认关）；后台任务本期用 FastAPI `BackgroundTasks`，Celery/ARQ 延后
- 前端：Vue 3 + Vite + TypeScript + Pinia + Naive UI + ECharts + Axios
- 认证：JWT
- 包管理：后端 `pip install -e ".[dev]"`；前端 `npm`

换栈需求一律先去 `08_决策记录.md` 走一遍。

## 分层约束（硬规则，违反即缺陷）

```
API (api/v1/*)         只做参数校验 + 调用 Service，禁止写 SQL / 操作 ORM
  ↓
Service (services/*)   业务编排 + 事务，所有 DB 写操作在这层
  ↓
Repository (repositories/*)   纯数据访问，不写业务
  ↓
Model (models/*)       SQLAlchemy ORM
```

AI 走独立竖线：
```
Service → ai/pipelines/* → ai/providers/*（LLMProvider 抽象）
                       ↘ ai/cache.py / ai/audit.py
```

**禁止**：业务代码里 `import openai` / 直接 HTTP 调模型 / 裸 `json.loads` 后落库 / 主流程 `if module_code == "xxx"` 硬编码。

## 后端目录（实际现状）

```
server/
├── app/
│   ├── main.py                 FastAPI 入口
│   ├── core/                   配置/安全/日志
│   ├── api/v1/                 auth / diaries / events / modules / profile / stats
│   ├── schemas/                Pydantic DTO
│   ├── models/                 user / diary / event / module / profile / ai_call_log / action_log
│   ├── repositories/           DAO
│   ├── services/               auth / diary / event / module / profile / stats
│   ├── ai/
│   │   ├── providers/          base / deepseek / ...
│   │   ├── prompts/            jinja2 模板
│   │   ├── pipelines/          diary_parser / action_executor
│   │   ├── cache.py            prompt 哈希缓存
│   │   └── audit.py            调用审计落 ai_call_log
│   ├── tasks/                  后台任务（当前 BackgroundTasks，未来 Celery）
│   ├── modules/                内置模块：weather / meal / expense / memo
│   │   ├── _base.py            模块基类
│   │   └── registry.py         @register 注册入口
│   └── db/
│       ├── session.py
│       └── migrations/         alembic
└── tests/
    ├── conftest.py
    ├── test_health.py
    └── test_modules.py
```

## 关键文件指针

| 目的 | 文件 |
|---|---|
| 新增业务模块 | `server/app/modules/<code>/` + `registry.py` `@register` |
| 新增 LLM Provider | `server/app/ai/providers/<name>.py` + 在 `providers/registry.py` 注册 |
| 改日记解析主流程 | `server/app/ai/pipelines/diary_parser.py` |
| 改 Action 执行 | `server/app/ai/pipelines/action_executor.py` |
| 配置项 | `server/app/core/config.py`（`Settings`） + `server/.env` |
| 新建表 | `server/app/models/*.py` + `alembic revision --autogenerate` |
