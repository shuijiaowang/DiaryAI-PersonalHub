# docs · 文档总入口

这个项目的文档分两类，按"读者是谁"严格区分：

## 🤖 `docs/ai/` — 给 AI 看

结构化、短、token 友好。AI（Cursor / Claude / ChatGPT）在本仓库干活时的"长期记忆"。

- 入口：[`docs/ai/README.md`](./ai/README.md)
- 内容索引：
  - `01_项目定位.md` · `02_架构与目录.md` · `03_数据模型.md`
  - `04_模块约定.md` · `05_核心流程.md`
  - `06_工作流.md` — AI 接到任务的 SOP
  - `07_进度与状态.md` — 当前进度 / WIP / bug（AI 动态维护）
  - `08_决策记录.md` — 重要技术决策留痕

## 👤 `docs/human/` — 给你（开发者）看

通俗、松散、"1 分钟找回项目感觉"。

- 入口：[`docs/human/README.md`](./human/README.md)
- 核心文件：
  - `本地运行指南.md` — 换机器 / 教别人跑
  - `本地环境速记.md` — 密钥 / 版本号 / 常用命令
  - `产品需求_通俗版.md` — 一页纸想起 MVP 在做什么
  - `数据库一览.md` · `框架逻辑图解.md`
  - `开发进度.md` · `下一步建议.md` · `测试体验与反馈.md`
  - `项目想法_原始.md` — 立项时的原始脑洞

## 🔒 硬约束（始终生效）

[`.cursor/rules/diaryai-rules.mdc`](../.cursor/rules/diaryai-rules.mdc) 是 AI 的硬规则，包括：

- LLM / 模块 / 服务分层 / 安全 / 测试 等红线
- **第 15~17 条**规定了 AI 每完成一个阶段性任务必须同步哪些文档

违反任意一条视为缺陷。

## 同一件事只在一处维护

- 数据库**字段级**细节 → 只在 `docs/ai/03_数据模型.md`；`docs/human/数据库一览.md` 只有人话概览。
- 进度的**详细任务态** → 只在 `docs/ai/07_进度与状态.md`；`docs/human/开发进度.md` 只有里程碑。
- 运行步骤 → 只在 `docs/human/本地运行指南.md`；`docs/ai/` 不重复。
